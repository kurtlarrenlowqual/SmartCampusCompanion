# 📱 Smart Campus Companion

## 📖 Description
**Smart Campus Companion** is a mobile application designed to assist university students by providing relevant information and support related to their enrolled department.  
The application is **progressively developed throughout the second semester**, following academic phases (**Prelim, Midterm, and Finals**).

The project emphasizes:
- Collaborative software development  
- Proper role distribution  
- Version control best practices using **Git** and **GitHub**

## 🔧 Git Workflow

The team follows a structured Git workflow to ensure code quality, consistency, and smooth collaboration:

### (For Prelims)
1. **Project Initialization**  
   The Team Leader initializes the project and GitHub repository.

2. **Branching Strategy**  
   A dedicated `development` branch is created for active feature development.

3. **Task Assignment**  
   Roles and development tasks are assigned to each team member.

4. **Quality Assurance Review**  
   QA checks the code before any commits or pushes are made.

5. **Commit & Push Management**  
   The Git Manager oversees commit messages, merges, and pushes to avoid conflicts.

6. **Project Monitoring**  
   The Team Leader monitors overall progress and ensures alignment with project goals.
